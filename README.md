# Дерево выражений (Expression Tree)

## Описание задачи

Программа преобразует математическое выражение из **постфиксной формы** в **дерево выражений**, где листья — операнды (целые числа), а внутренние узлы — операторы `+`, `-`, `*`, `/`. После построения дерева значение выражения вычисляется рекурсивно.

### Пример

**Постфиксная запись:** `2 5 * 3 +`

**Дерево:**
/ \

3
/
2 5

text

**Результат:** `(2 * 5) + 3 = 13`

---

## Реализованные функции

| Функция | Назначение |
|---------|------------|
| `build_expression_tree(tokens)` | Построение дерева по списку токенов (постфиксная запись) |
| `evaluate(root)` | Рекурсивное вычисление значения выражения и подсчёт количества операций |

---

## Как это работает

### Построение дерева (алгоритм)

1. Создаётся пустой стек
2. Каждый токен обрабатывается:
   - **Число** → узел помещается в стек
   - **Оператор** → из стека извлекаются два узла (правый и левый операнд), создаётся узел-оператор с подвешенными потомками, результат помещается в стек
3. После обработки всех токенов в стеке остаётся корень дерева

### Вычисление значения

- **Лист** (число) → возвращает своё значение (0 операций)
- **Внутренний узел** (оператор) → рекурсивно вычисляет левое и правое поддеревья, применяет оператор, возвращает результат и общее количество операций

---

## Примеры работы

### Пример 1: `2 5 * 3 +`
Вход: 2 5 * 3 +
Результат: 13
Операций: 2



### Пример 2: `2 3 + 4 5 * +`
Вход: 2 3 + 4 5 * +
Результат: (2+3) + (4*5) = 25
Операций: 3
